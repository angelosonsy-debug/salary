# مجمّع مرتبات السرايا (Military Salary Aggregator)

تطبيق سطح مكتب (Windows) يقوم بقراءة ملفات Word الخاصة بمرتبات كل سرية، واستخراج
عمود **"الصافي"** فقط، وتجميع المبالغ المتشابهة، وحساب إجمالي كل سرية والإجمالي
العام، مع إمكانية التصدير إلى Excel / PDF / Word / CSV.

---

## 1. المتطلبات قبل التشغيل

يجب تثبيت **Node.js** (نسخة 20 أو أحدث) على جهازك:
https://nodejs.org

تأكد من التثبيت بكتابة الأمرين التاليين في Terminal / CMD:

```bash
node -v
npm -v
```

---

## 2. تثبيت المشروع (مرة واحدة فقط)

افتح Terminal داخل مجلد المشروع ثم نفّذ:

```bash
npm install
```

هذا الأمر يقوم بتحميل جميع المكتبات المطلوبة (React، Mammoth لقراءة ملفات
Word، مكتبات التصدير، Electron... إلخ).

---

## 3. التشغيل محلياً أثناء التطوير (Live / Hot Reload)

### أ) تشغيل كصفحة ويب فقط (أسرع للتجربة والتعديل):

```bash
npm run dev
```

سيظهر رابط مثل `http://localhost:5173` — افتحه في المتصفح. أي تعديل في الكود
ينعكس فوراً على الصفحة (Hot Reload).

### ب) تشغيل كتطبيق Desktop حقيقي (نافذة Electron) أثناء التطوير:

```bash
npm run electron:dev
```

سيتم تشغيل خادم Vite ثم فتح نافذة Electron تعرض التطبيق مباشرة، مع أدوات
المطوّر (DevTools) مفتوحة تلقائياً للمساعدة في تتبع أي مشاكل.

---

## 4. بناء نسخة الإنتاج (Live / جاهزة للنشر)

### أ) بناء ملفات الويب النهائية فقط:

```bash
npm run build
```

تُنتج مجلد `dist/` يحتوي على التطبيق كملفات HTML/CSS/JS جاهزة، يمكنك معاينتها
محلياً بالأمر:

```bash
npm run preview
```

> **ملاحظة توافق ويندوز 8 / 8.1:** تم تثبيت نسخة Electron على `22.3.27`
> عمداً (بدلاً من أحدث نسخة)، لأن Electron ابتداءً من الإصدار 23 توقف عن
> دعم ويندوز 7 و8 و8.1 نهائياً (يتطلب ويندوز 10 فأعلى). النسخة 22 هي آخر
> نسخة رسمية تعمل على ويندوز 8/8.1، ويتم بناء نسختين من المثبّت (32 بت
> و64 بت) ليعمل على الأجهزة القديمة أيضاً. العيب الوحيد: هذه النسخة من
> Electron لم تعد تصلها تحديثات أمنية من الشركة المصنّعة — وهذا أمر
> مقبول لتطبيق داخلي غير متصل بالإنترنت مثل هذا.

### ب) بناء ملف تثبيت Windows (.exe):

```bash
npm run electron:build
```

هذا الأمر يقوم بـ:
1. بناء نسخة الإنتاج من الواجهة (`vite build`).
2. تغليفها داخل تطبيق Electron وإنتاج ملف تثبيت Windows (NSIS installer).

الناتج يكون داخل مجلد **`release/`** — ستجد **ملفّي** تثبيت (لأن البناء الآن
ينتج نسختين لتغطية أكبر عدد من الأجهزة، بما فيها أجهزة ويندوز 8/8.1 ذات
المعالج 32 بت):

```
release/مجمع مرتبات السرايا Setup 1.0.0 x64.exe   (لأغلب الأجهزة الحديثة)
release/مجمع مرتبات السرايا Setup 1.0.0 ia32.exe   (للأجهزة القديمة 32 بت)
```

هذا هو ملف التثبيت الذي يمكنك تشغيله على أي جهاز Windows، وسيثبّت التطبيق
كأي برنامج عادي (أيقونة على سطح المكتب/قائمة ابدأ).

> ملاحظة: إذا كنت تبني الملف من نظام غير Windows (مثل Linux أو Mac)، قد
> يحتاج `electron-builder` لتثبيت أدوات إضافية أول مرة (`wine` مثلاً على
> لينكس) لإنتاج ملف NSIS الخاص بـ Windows. الأسهل هو تنفيذ هذا الأمر مباشرة
> على جهاز Windows نفسه.

### ج) بناء سريع بدون مثبّت (للتجربة فقط):

```bash
npm run electron:build:dir
```

يُنتج مجلد يحتوي على `.exe` قابل للتشغيل مباشرة بدون معالج تثبيت — مفيد
للتجربة السريعة.

## 5. تشغيل الاختبارات (Unit Tests)

```bash
npm test
```

يغطي الاختبارات:
- دقة تحويل النصوص إلى أرقام (بما فيها الأرقام العربية والفواصل العشرية).
- دقة التجميع (Group By) وعدم فقد القروش نتيجة التقريب.
- استخراج عمود "الصافي" فقط من الجدول وتجاهل الأسماء والتوقيعات وصف الإجمالي.
- التعامل مع الملفات غير الصحيحة (عمود غير موجود، ملف فارغ، لا يوجد جدول).

---

## 6. بنية المشروع (Clean Architecture)

```
src/
  core/
    types.ts                  # الأنواع المشتركة بين كل الطبقات
    money.ts                  # تحويل/تنسيق المبالغ بدقة كاملة (بدون Float)
    parser/docxParser.ts      # قراءة ملف Word واستخراج عمود "الصافي"
    aggregator/aggregator.ts  # تجميع المبالغ المتكررة (Group By)
    report/reportGenerator.ts # تنسيق تقرير كل سرية + التقرير العام
    export/exportService.ts   # التصدير إلى Excel / CSV / PDF / Word
  components/                 # واجهة المستخدم (React)
  styles/app.css              # التنسيق (Cairo + RTL + Dark/Light Mode)
electron/
  main.cjs                    # نافذة سطح المكتب (Electron)
```

كل طبقة مستقلة تماماً عن الأخرى، بحيث يمكن مستقبلاً:
- إضافة قارئ PDF أو OCR بدون التأثير على باقي الكود (فقط طبقة `parser`).
- تغيير طريقة التصدير أو إضافة صيغ جديدة (فقط طبقة `export`).
- ربط قاعدة بيانات لحفظ سجل شهري ومقارنة الشهور (طبقة جديدة `report`).

---

## 7. طريقة عمل الاستخراج والتجميع (باختصار)

1. يقرأ التطبيق كل ملف Word ويحوّله لجدول HTML داخلياً.
2. يبحث عن عمود عنوانه "الصافي" (أو أي صيغة مشابهة له).
3. يستخرج كل القيم الرقمية من هذا العمود فقط، ويتجاهل الأسماء والتوقيعات
   وصف "الإجمالي" في آخر الجدول.
4. يحوّل كل قيمة إلى عدد صحيح بالقروش (مثلاً 331.37 جنيه = 33137 قرش) لضمان
   عدم حدوث أي خطأ تقريب عند جمع آلاف القيم.
5. يجمع (Group By) القيم المتطابقة ويحسب: القيمة × العدد = الإجمالي.
6. يكرر هذا لكل ملف، ثم يحسب الإجمالي العام لجميع السرايا.

---

## 8. ملاحظات مهمة

- التطبيق يدعم حالياً ملفات **.docx** فقط (حسب المطلوب). دعم PDF/OCR مُهيأ
  في البنية لكن غير مفعّل بعد — يمكن إضافته لاحقاً في طبقة `parser` فقط.
- إذا لم يُعثر على عمود "الصافي" في ملف معيّن، يظهر تحذير واضح باسم الملف
  المتسبب في المشكلة بدلاً من إيقاف تحليل باقي الملفات.
- الوضع الليلي/النهاري يُحفظ تلقائياً حتى بعد إغلاق التطبيق.
